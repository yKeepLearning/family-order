# 家庭厨房 — 角色选择与家庭加入流程重设计

## 概述

重构首页角色选择与家庭绑定流程。用户选择角色（主厨/吃货）后，通过底部弹层创建或加入家庭，家庭以唯一名称作为标识。

## 流程

```
首页(角色选择) ──选「主厨」──→ 底部弹层：创建家庭 ──提交──→ 主厨面板
     │                              │
     └──选「吃货」──→ 底部弹层：加入家庭 ──提交──→ 家庭菜单
```

## 页面设计

### 1. 首页（role-selection）

在现有设计基础上微调：

- **保留**：顶部 🍳 图标 + 「家庭厨房」标题 + 「今天想吃什么？」副标题，中间双卡片布局
- **改动**：角色标签「家人」→「吃货」，卡片点击改为触发底部弹层（不再直接跳转）
- **底部文案**：保持「无需注册，选择角色即可开始」

### 2. 底部弹层·主厨版（创建家庭）

- 半屏高度 ~300px，圆角 20px，白色背景
- 标题：「创建我的厨房」+ 👨‍🍳 图标
- 输入框：家庭名称，placeholder "取个名字，比如'张家厨房'"
- 提示文案：「名称唯一，创建后不可修改」
- 按钮：橙色主按钮「创建并进入」
- 关闭方式：左上 × 或下拉手势
- **错误状态**：名称已被占用 → 输入框下方红字「该名称已被使用，换一个试试」

### 3. 底部弹层·吃货版（加入家庭）

- 与主厨版结构一致
- 标题：「加入家庭」+ 🍽️ 图标
- 输入框：家庭名称，placeholder "输入家庭名称"
- 提示文案：「输入主厨创建的家庭名称即可加入」
- 按钮：蓝色按钮「加入」（区别于主厨橙色）
- **错误状态**：家庭不存在 → 输入框下方红字「未找到该家庭，请确认名称」

## 数据模型

### 集合: `families`

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | string | 自动生成 |
| `name` | string | 家庭名称，唯一索引 |
| `chefId` | string | 主厨 openid |
| `createTime` | Date | 创建时间 |

### 集合: `users`（扩展）

在现有字段上新增：

| 字段 | 类型 | 说明 |
|------|------|------|
| `role` | 'chef' \| 'family' | 角色 |
| `familyName` | string | 所属家庭名称 |

## 校验规则

- 家庭名称：2-20 字符，不可为空
- 主厨创建时需先登录，每个用户只能创建一个家庭
- 吃货可加入任意已存在的家庭
