# 家庭厨房助手 - 产品设计文档

## 1. 项目概述

一个极简、零支付的家庭内部点餐微信小程序。让"今天吃什么"的决策变得有趣，让"做饭"变成明确的任务响应。

### 核心用户角色

- **厨师 (Chef)**：掌勺人，发布菜品、管理菜单、接单、出餐
- **家人 (Family)**：其他家庭成员，浏览菜单、加购物车、提交订单

### MVP 范围

- 家人端：完整点餐体验（菜单浏览、购物车、下单、订单追踪）
- 简约厨师端：接单/出餐流程、菜单上下架管理

## 2. 页面设计

共 6 个页面，设计文件位于 `family-kitchen.pen`。

### 2.1 启动页 / 角色选择 (1-role-selection)

- **布局**：竖屏 390×844，米白背景
- **状态栏**：暖橙 #FF6B35
- **品牌区**：🍳 图标 + "家庭厨房" + "今天想吃什么？"
- **角色选择**：两个大卡片并排
  - 👨‍🍳 厨师（暖橙背景）
  - 👨‍👩‍👧‍👦 家人（深灰背景）
- **底部提示**："无需注册，选择角色即可开始"

### 2.2 家人端 - 今日菜单 (2-family-menu)

**参照沃尔玛/永辉线上超市布局**

- **布局**：竖屏 390×844
- **顶部区域**：👋 今天想吃什么？ + 厨师营业状态
- **主内容区**：左右分栏
  - **左侧分类导航**（宽 85px）：
    - 主食 🍚（选中态：白底 + 橙色高亮）
    - 汤品 🍜
    - 小食 🥗
    - 甜品 🍰
  - **右侧菜品列表**（剩余宽度）：
    - 按分类分组，每组有标题分隔
    - 每行一个菜品：缩略图（64×64）+ 名称 + 描述 + [+] 添加按钮
- **底部悬浮购物车栏**（深灰圆角胶囊）：
  - 🛒 图标 + 总价 + 角标 | [去下单] 按钮

### 2.3 购物车弹窗 (3-cart)

- **布局**：底部弹出卡片，390×520，圆角 20
- **拖拽手柄**
- **标题**："我的订单" + "清空"
- **菜品列表**：
  - 每行：图标 + 名称 + 数量控件（− 1 +）
- **分隔线**
- **备注输入**："特殊要求" → "例如：少盐、多辣..."
- **底部**：合计 ¥0 + [下单] 按钮

### 2.4 我的订单 (4-my-orders)

- **布局**：竖屏 390×844
- **标题**："我的订单" + "追踪你的餐食进度"
- **订单卡片列表**：
  - 每单显示：图标 + 菜品摘要 + 状态徽章 + 时间 + 备注
  - 状态：待接单（灰）、烹饪中（黄）、已出餐（绿）
- **底部 Tab 栏**：菜单 | 订单

### 2.5 厨师端 - 订单看板 (5-chef-board)

- **布局**：竖屏 390×844
- **标题**："订单看板" + "3 个待处理 | 厨房营业中"
- **状态过滤标签**：待接单 | 烹饪中 | 已出餐（选中态高亮）
- **订单卡片**：
  - 谁点的 + 时间
  - 菜品明细 + 备注
  - 操作按钮：[接单] → [完成]（状态流转）
- **底部 Tab 栏**：订单 | 菜单

### 2.6 厨师端 - 菜单管理 (6-chef-menu)

- **布局**：竖屏 390×844
- **标题**："菜单管理" + "管理今日供应"
- **厨房开关**："厨房营业中"（绿色切换开关）
- **菜品列表**：
  - 每行：图标 + 名称 + 分类 + 上下架开关（橙色=上架，灰色=下架）
- **[+ 添加菜品]** 按钮
- **底部 Tab 栏**：订单 | 菜单

## 4. 数据模型

### users 集合
- `_id`: string
- `nickName`: string
- `avatarUrl`: string
- `role`: 'chef' | 'family'
- `familyId`: string

### dishes 集合
- `_id`: string
- `name`: string
- `image`: string
- `category`: string
- `isAvailable`: boolean
- `familyId`: string

### orders 集合
- `_id`: string
- `familyId`: string
- `dinerId`: string
- `dinerName`: string
- `items`: array [{ dishId, dishName, quantity }]
- `notes`: string
- `status`: 'pending' | 'cooking' | 'ready'
- `createTime`: Date

## 5. 技术选型

- **平台**：微信小程序原生 + 微信云开发 (CloudBase)
- **UI 组件库**：TDesign 小程序组件库
- **云函数**：
  - `getMenu`：获取今日菜单
  - `submitOrder`：创建订单
  - `updateOrder`：厨师更新订单状态
  - `manageDish`：增删改菜品
- **实时性**：使用数据库 watch 监听订单变更
